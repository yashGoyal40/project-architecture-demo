AWS Infrastructure Architecture

This diagram illustrates the comprehensive AWS infrastructure that powered our project. It showcases the intricate network of services and components that worked together to deliver a robust and scalable solution.

Key components include:

  • VPC for isolated network environments
  • Subnets for public and private resource allocation
  • NAT Gateway for private subnet internet access
  • Internet Gateway for public subnet internet connectivity
  • Elastic Container Registry (ECR) for storing container images
  • Elastic Container Service (ECS) for container orchestration
  • Application Load Balancer (ALB) for traffic routing and HTTPS redirection
  • Security Groups for inbound and outbound traffic control
  • Auto-scaling for ECS task management based on resource usage
  • Ansible Playbooks for automated infrastructure provisioning
  • GitHub Actions CI/CD for seamless code deployment
  • MongoDB Atlas for external database hosting
  • AWS Secrets Manager for secure credential management
  • Node.js backend for job portal functionality
  • Docker for application containerization

This architecture was designed to handle high traffic loads while maintaining optimal performance and security.
